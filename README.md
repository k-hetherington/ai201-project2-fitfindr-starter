## The Mock Listings Dataset

`data/listings.json` contains 40 mock secondhand listings across categories (tops, bottoms, outerwear, shoes, accessories) and styles (vintage, y2k, grunge, cottagecore, streetwear, and more).

Each listing has: `id`, `title`, `description`, `category`, `style_tags`, `size`, `condition`, `price`, `colors`, `brand`, and `platform`.

Load it with:
```python
from utils.data_loader import load_listings
listings = load_listings()
```

## The Wardrobe Schema

`data/wardrobe_schema.json` defines the format your agent uses to represent a user's existing wardrobe. It includes:

- `schema`: field definitions for a wardrobe item
- `example_wardrobe`: a sample wardrobe with 10 items you can use for testing
- `empty_wardrobe`: a starting template for a new user

Load an example wardrobe with:
```python

wardrobe = get_example_wardrobe()
```

## Tool Inventory

### Tool 1: search_listings(description, size, max_price)

**Purpose:** Searches the mock secondhand listings dataset and returns matching items based on the user's description, size, and maximum price.

**Inputs:**

* description (str): Keywords describing the item the user wants.
* size (str | None): Optional size filter.
* max_price (float | None): Optional maximum price filter.

**Returns:**

* list[dict]: A list of matching listing dictionaries sorted by relevance. Each listing contains id, title, description, category, style_tags, size, condition, price, colors, brand, and platform.

### Tool 2: suggest_outfit(new_item, wardrobe)

**Purpose:** Generates outfit suggestions using a selected thrifted item and the user's wardrobe.

**Inputs:**

* new_item (dict): The selected listing from search_listings().
* wardrobe (dict): The user's wardrobe containing an items list.

**Returns:**

* str: One or more outfit suggestions. If the wardrobe is empty, the tool provides general styling advice instead.

### Tool 3: create_fit_card(outfit, new_item)

**Purpose:** Creates a short social media style caption describing the outfit and thrifted item.

**Inputs:**

* outfit (str): Outfit suggestion returned by suggest_outfit().
* new_item (dict): The selected thrifted listing.

**Returns:**

* str: A caption-style fit card. If the outfit is empty, returns a descriptive error message.

## Planning Loop

The agent follows a conditional planning loop:

1. Parse the user query to extract a description, size, and maximum price.
2. Call search_listings() using the parsed values.
3. If no listings are found, store an error message in the session and stop the workflow.
4. Select the top listing from the search results and store it as selected_item.
5. Call suggest_outfit() using the selected item and the user's wardrobe.
6. Store the outfit suggestion in the session.
7. Call create_fit_card() using the outfit suggestion and selected item.
8. Store the fit card in the session and return the completed session.

## State Management

The agent uses a session dictionary to store information throughout a user interaction. This allows information from one tool to be passed to the next without requiring the user to re-enter it.

The following values are stored in the session:

- parsed: extracted query parameters (description, size, max_price)
- search_results: list of matching listings returned by search_listings()
- selected_item: the top listing chosen from the search results
- wardrobe: the user's wardrobe data
- outfit_suggestion: text returned by suggest_outfit()
- fit_card: caption generated by create_fit_card()
- error: error message if the workflow ends early

This state management approach allows the selected item from search_listings() to be reused by suggest_outfit() and create_fit_card() during the same interaction.

- Tool: search_listings()
- Input: description = "vintage graphic tee", size = None, max_price = 30

**Step 2 — Tool called:**
- Tool: suggest_outfit()

- Tool: create_fit_card()
- Input: outfit suggestion from Step 2, selected item from Step 1

- Output: Social media style fit card.

User receives:

- The top thrift listing found.
- Outfit recommendations.
- A shareable fit card caption.

## Error Handling and Fail Points

<!-- For each tool, describe the specific failure mode and what your agent does in response.
     This maps to the error handling section of the rubric (F5-C1). -->

| Tool | Failure mode | Agent response |
|------|-------------|----------------|
| `search_listings` | No results match the query |Stores an error in session["error"], returns early, and suggests broadening the search criteria.|
| `suggest_outfit` | Wardrobe is empty | Generates general styling advice instead of referencing wardrobe items. |
| `create_fit_card` | Outfit input is missing or empty | Returns a descriptive error message instead of crashing. |

In addition, I tested `designer ballgown size XXS under $5`, which returned no search results. The agent displayed: "I couldn't find any listings that matched that exact request. Try using a broader description, increasing your budget, or removing the size filter." It then stopped before calling `suggest_outfit()` or `create_fit_card()`.
